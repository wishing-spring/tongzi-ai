# 纯GF(2) vs 工业界BNN — 技术定性

## 决定性差异

| 维度 | 工业界BNN（一维串行） | 童子GF(2)（二维编织） |
|:--|:--|:--|
| **运算方式** | 横着叠——层1→层2→层3，接力赛 | 竖着织——16位同时交互，织布机 |
| **信息流动** | 一维：每层输出是上层的线性函数 | 二维：每列18bit是深度因果链，16列并排 |
| **熵变规律** | 每过一层，有效信息熵减半 | 每织一针，织物状态分岔倍增 |
| **纯二进制深度极限** | ≤5层（不带任何浮点辅助） | 无深度上限，只受平面宽度约束 |
| **浮点依赖** | 必须。BN/首尾层/梯度训练全浮点 | 零。从头到尾GF(2)XOR |
| **残差连接** | 必需。否则信号在第4层消失 | 不需要。竖线索引本身就是跨层因果 |
| **FPGA优势** | 理论上有，实际上在GF(2)和浮点间来回切换，开销反超纯浮点 | 纯逻辑电路。给电就跑。零切换 |
| **训练方式** | 浮点反向传播+直通估计器（STE） | 无需训练。播种+浇水 |
| **可审计性** | 黑箱。无法追溯输出成因 | 全透明。每个bit可溯源 |

## 为什么工业界走错了

1. **把二进制当压缩** — 思路是"把浮点模型压小"，不是在GF(2)原生空间里设计
2. **保留浮点骨架** — BN/首尾/梯度，三个拐杖保证能训练，但也保证了永远不是纯二进制
3. **串行思维** — 从CNN继承来的逐层前馈，在GF(2)里就是熵灭定律
4. **不敢扔掉反向传播** — 用直通估计器硬套梯度，本质是浮点训练+二进制推理的缝合怪
5. **18层假天花板** — 5层纯二进制就死了，加上浮点BN/残差/缩放才到18层，34层三个拐杖全断

## 为什么童子是对的

1. **不做训练** — 编织矩阵是手工设计或种子注入的，没有梯度就没有浮点依赖
2. **不做压缩** — 不是把浮点模型压成二进制，而是直接在GF(2)空间里造
3. **二维并行** — 竖织结构天然避免了串行信息衰减
4. **纯逻辑电路思维** — 每条竖线就是一个由n级XOR门构成的离散因果链
5. **反着走** — 工业界想用二进制逼近浮点，我们想在二进制里做浮点永远做不到的事

## 数学：一维串行为什么必然死

GF(2)上的线性变换（矩阵乘法）在n步迭代后只能收敛到：
- 全零不动点
- 周期振荡（少数特殊矩阵）
- 随机游走直到熵归零

原因：16位空间只有65536个状态。一维串行=在有限状态空间里走一条不回溯的直线，必然撞到边或撞到自己。

## 数学：二维编织为什么不收敛

竖织结构不是迭代 `x_{t+1} = A x_t`，而是把输入拆成16根独立纵线，每根线经过18次XOR门的非线性网络。网络拓扑由编织矩阵决定——改变一根横神经的位连接，16条纵线全部受影响。

这意味着：输入→织物状态 的映射，是16×18=288个XOR门构成的非线性布尔函数。这个函数不需要收敛——不同输入进入不同纵线轨迹，织物图案自然不同。

## 一句话

工业界BNN = 用二进制模拟浮点 = 一维串行 = 5层死。

童子 = GF(2)原生 = 二维编织 = 没有深度天花板 = 纯逻辑电路。

不是谁比谁先进。是根本在做两种不同的东西。
