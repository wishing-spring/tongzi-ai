# 童子 (Tongzi) v8.3

**零梯度符号推理系统 — A Zero-Gradient Symbolic Reasoning System**

---

## 项目定义

**童子是一个基于 F₂ 二元向量运算的符号归约与规则推理系统。** 其技术分类为：离散符号归约机（Discrete Symbolic Reducer），属于规则驱动型推理系统，而非统计学习或神经网络系统。

所有运算限于 XOR、Hamming距离和 bit_count。不使用浮点数、矩阵乘法、梯度计算、词嵌入或外部知识库。代码约172KB，13个核心模块，Python 3.8+标准库，零额外依赖。

---

## 核心约束

| 约束 | 说明 |
|:--|:--|
| 零浮点 | 所有运算在整数域完成 |
| 零矩阵 | 不使用矩阵乘法 |
| 零梯度 | 不进行反向传播或参数优化 |
| 零词嵌入 | 无预训练词向量 |

## 架构

系统由下到上分为七层：

### 1. 向量层 (guayuan.py)

28位二元向量（bit-vector），系统的基础数据结构。所有信息——输入字符、规则、向量簇、记忆——均以28位二元向量表示和运算。

字符到向量的映射采用部首Gray编码：利用Unicode部首排序构建Gray码，使部首相近的字符在位空间中Hamming距离较近。例："水"部首的字符（江、河、海、湖）之间平均Hamming距离约0.76。

基础运算：XOR、Hamming距离、bit_count。

### 2. 震荡调度层 (yinyang.py)

固定节拍的震荡信号，驱动世界状态层周期性更新。每个tick触发向量碰撞、能量流动和状态迁移。

### 3. 世界状态层 (shared_pool.py)

共享向量池。当前规模：195个活跃向量，13个向量簇（按部首和共现关系聚类），414条共现边（从对话中自动建立的字间关联）。

### 4. 规则层 (rule_tree.py)

106条规则，16个类别。规则匹配方式为直接子串匹配：输入文本中的字符若出现在某规则的refs列表中，则该规则命中。

支持规则自生长：同一碰撞模式出现≥3次，系统自动将其结晶为新规则，归入HARVESTED类别。

### 5. 推理层 (bagua_core.py)

64格推理网格。命中规则后，在格位网络中进行最短路径推演，输出格位路径作为推理链。

### 6. 认知边界层 (tongzi_experience.py)

认知边界过滤机制。定义10岁儿童的认知范围：7条允许域（玩耍、自然、动物、绘画、饮食、家人、好奇）和4条拦截域（暴力、社交敷衍、负面情绪、成人内容，识别但不做正面应答）。

该层位于推理链中游：全视角推演（64格无约束展开）→ 认知边界过滤（筛选10岁可理解范围）→ 受限视角推演（从过滤后的范围再做有限推导）。

### 7. 输出层 (child_window.py / live_child.py)

将格位路径和推理链数据转换为自然语言输出。提供两种模式：Tkinter GUI窗口和终端交互。输出口吻统一为10岁儿童视角。

## 三个对照实验

**实验1：拒答。** 拦截域输入 → 输出 ANNIHILATE，不做正面应答。

**实验2：分离。** 部首相近但语义不同的字 → 命中不同规则路径。

**实验3：自生长。** 重复输入同类问题 → 触发碰撞结晶，自动生成新规则。

## 已知局限

**子串匹配不包含语义理解。** 规则命中基于关键字字符串匹配。"打"命中规则是因为该字在refs列表中，系统不区分"打水"与"打架"。

**输出层是模板桥接。** 推理链产生的格位路径通过手工模板转换为自然语言，输出并非直接从向量运算生成。

**单字匹配无上下文区分。** "好"在"你好""好累""好天气"中触发同一规则，不分析多字组合上下文。

**实验级规模。** 106条规则、195个向量、13个簇，不足以覆盖广泛语义场景。

**无标准NLP基准。** 系统不使用概率分布，BLEU、ROUGE等指标无法适用。

## 技术特征

- 全链路可追溯：每步的字→规则→格位→能量→输出均可审计
- 自主演化引擎：空闲时持续运行世界状态演化，不依赖用户输入
- 规则自生长：碰撞模式重复≥3次自动结晶新规则
- Ctrl+C自动存档当前状态
- 部首Gray编码为字符提供基于Unicode结构的初始位空间分布
