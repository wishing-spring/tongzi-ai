# 评审者指南 · Tongzi v4 F₂ 底层

**你好，感谢你花时间看这个项目。**

## 这是什么

一个 **纯 F₂ 空间（GF(2)上的离散向量空间）的另类 AI 底层实验**。

不是 LLM、不是神经网络、不是 token 预测器。是一套基于 XOR/AND 碰撞的离散动力系统。

## 怎么跑

```bash
git clone https://github.com/wishing-spring/tongzi-ai.git
cd tongzi-ai
python demo.py
```

没有依赖。`pip install nothing`。

## 看什么

`demo.py` 六个步骤展示完整链路：

| 步骤 | 看什么 |
|:--|:--|
| 1. 编码 | 汉字 → 四芯弹珠 → 80个28bit F₂向量 |
| 2. 涌动池 | 80卦进池，准备碰撞 |
| 3. 碰撞300 tick | XOR+AND 生子，涌动窗口滑动，反熵微扰 |
| 4. 吸引子 | 固化子代 express 回原字 → 吸引子分布 |
| 5. 碰撞链 | 单个 XOR 碰撞的 bit 级追踪 |
| 6. 对话对比 | 不同输入产生不同吸引子分布 |

## 诚实说明

**输出层（speak.py）是模板桥接，不是 F₂ 生成。**

F₂ 碰撞产出的是 28bit 的位向量和吸引子分布。从位向量到自然语言的映射，目前靠人工模板。这是当前最大的架构限制——我们诚实标注，不假装 F₂ 能生成中文。

真正值得我们骄傲的东西在底层：**没用任何浮点数、矩阵、梯度，20个汉字的 F₂ 碰撞产生了可区分的吸引子结构。** 输入不同 → 吸引子不同 → 回应不同。

## 我们想确认的

1. F₂ 碰撞引擎的设计是否有根本性错误
2. 吸引子涌现是否是真正的结构或只是编码假象
3. 这个方向是否值得继续

任何反馈都欢迎。包括"这方向没戏"。
