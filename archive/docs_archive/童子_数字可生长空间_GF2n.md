# 童子 · 数字可生长空间 — GF(2)ⁿ 软骨架

**定位**: 童子唯一底层数学模型 | **状态**: 暂定
**替代**: GF(2)³（已弃——维度过硬，锁死上限）

---

## 核心概念

```
GF(2)ⁿ
  n 不预设。不锁死。
  n 随输入密度自动生长。

初始：n 小 → 分辨粗糙
输入越多 → 空间不够 → 自发生出新维度 → n+1
万亿输入 → n 极大 → 空间极密 → 极稳
```

---

## 越砸越稳的原理

```
稀疏空间：点散落，跳转远，一扰动就飘
密实空间：点密集，邻居近，跳不动远路

万亿输入 = 万亿点铺在 GF(2)ⁿ 里
→ 局部密度极高
→ 每个簇都被邻近簇"夹稳"
→ 扰动打不散
```

## 六性在GF(2)ⁿ里的位置

六性不变。六性是GF(2)ⁿ上的**基本操作规则**：

| 六性 | GF(2)ⁿ中 |
|:--|:--|
| 汉明距离 | n维空间中的距离度量 |
| 逐位取反 | n维对极点 |
| 格雷码 | n维超立方体上的哈密顿环 |
| 1位计数 | n维空间层级划分 |
| XOR | n维向量加法 |
| 邻接 | n维超立方体的边 |

> 六性不锁死n。六性在任意n下都成立。

## 生长触发

```
什么时候n长一位？
→ 当前空间内簇态密度超过阈值
→ 无法用现有n位区分新输入
→ 自动展开第n+1位
→ 空间容量×2
```

## 工程落地：Loom编织引擎（2026-05-20）

GF(2)ⁿ的理论路径在代码中通过 **Loom** 实现：

```
16位输入 → 18层编织矩阵 → 16位织物输出
每输出位 = XOR(选定bits) ^ AND(两输入bits) ^ ref_bit
每层后加逐层异化S盒置乱
```

### 为什么必须四重混合

| 缺失层 | 后果 | 实证 |
|:--|:--|:--|
| 纯XOR | 5层内坍缩为不动点 | 5/5碰撞 |
| XOR+S盒 | 固定点收敛 | 100→1 |
| XOR+AND(无ref) | 深度增加仍塌 | 29/200 |
| XOR+AND+S盒+ref | 无坍缩 | 200/200唯一 |

### 竖贴升维：n的生长路径

n不连续生长。n=16是"最小智能单元"——刚好形成一个完整的自洽计算闭环。

升维方式：**竖贴**，非横叠。

| | 横叠（串行叠加） | 竖贴（并行扩展） |
|:--|:--|:--|
| 操作 | 旧层上盖新层 | 旧比特旁贴新比特 |
| 接口 | 有强制压缩接口 | 无接口，同一平面 |
| 信息 | 每层熵减半 | 永不丢失 |
| 升级 | 推倒重训 | 改VEC_DIM常量 |

n=16→32→64→128… 每次翻倍，旧知识100%保留。
