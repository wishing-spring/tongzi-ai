```
│ φ 母体 (PHI_BITS)                    │
│ 世界: 🌿 生物                        │
│ 输入: — (系统常量)                    │
│ 输出: 256-bit 位串                    │
```

### 04 · 弹珠机编码

```
┌──────────────────────────────────────┐
│ 弹珠机编码 (Pinball Encoding)         │
│ 世界: 🔧 机械 → 💻 代码               │
│ 输入: Unicode码点 (16-bit)             │
│ 输出: Gua (32-bit F₂值)               │
│ 依赖: 柱阵常量 (当前: 质数公式)        │
│ 状态: ✅ 在用 (v2.3核心)              │
│ 理论: 逐位选柱XOR累加, 非线性F₂保距    │
│       bit差≈路径差→结果差             │
│ 引用: tongzi.py _pinball(), _encode_A/B/C32 │
│ 位置: 编码层                          │
│ 参数: 16柱 × 3台 = 48个32位常量        │
│ 论文: doc/弹珠机编码.md               │
└──────────────────────────────────────┘
```

**作用**：汉字→数字。核心价值：近邻码点→近邻卦元。纯位运算。

**当前柱阵来源**：质数公式 `(i*2654435761^(i+1)*16777619)`。应改为从 φ 母体切片生成。

**输出细节**：32位 = 低16位(XOR累加结果) | (低16位<<16)。镜像填充。

### 05 · 三轴 TMR

```
┌──────────────────────────────────────┐
│ 三轴TMR (Triple Modular Redundancy)   │
│ 世界: 🔧 机械 → 💻 代码               │
│ 输入: 一字 → 三个Gua (G_A,G_B,G_C)    │
│ 输出: 三轴匹配 (ok_a, ok_b, ok_c)     │
│ 依赖: 弹珠机编码 (04), 位锁定(08)     │
│ 状态: ✅ 在用                         │

│       三轴全过才锁 → 航天级冗余       │

```

**作用**：三台弹珠机，三组不同柱子。每轴独立判断。三票全过才算是"锁"。防止单轴误判。

### 06 · 榫卯 (凸凹接触)

```
┌──────────────────────────────────────┐
│ 榫卯 (_fit)                          │
│ 世界: 🔧 机械                        │
│ 输入: 两个Gua (a, b)                  │
│ 输出: bool (a凸位 ∩ b凹位 ≠ 空)       │
│ 依赖: —                              │
│ 状态: ⚠️ 在用但残缺 (只看低8位)        │
│ 理论: 凸(1)插入凹(0)→接触成立         │
│       物理榫卯→F₂位运算               │
│ 引用: tongzi_kernel.py _fit() L106   │
│ 位置: 碰撞层                          │
│ 参数: 当前只取 pa = a.value & 0xFF    │
│       浪费了32位中的24位              │
└──────────────────────────────────────┘
```

**作用**：判断两个卦元能不能"插进去"。比"像不像"更物理——是一个的凸起插入另一个的凹陷。

**问题**：`pa = a.value & 0xFF` 只看低 8 位。32 位卦元浪费 75%。

### 07 · 位锁定阈值

```
┌──────────────────────────────────────┐
│ 位锁定阈值 (FIT_MIN)                  │
│ 世界: 💻 代码                        │
│ 输入: 两个Gua, 阈值m                  │
│ 输出: bool (a凸位≥m)                  │
│ 依赖: 榫卯(06)                        │
│ 状态: ✅ 在用, FIT_MIN=9              │
│ 理论: popcount(a.value & ~b.value)≥m │
│       至少m个凸位插入凹位→锁定         │
│ 引用: tongzi.py _fit_32()            │
│ 位置: 判决层                          │

│       三轴同时≥9 → 锁                 │
└──────────────────────────────────────┘
```

**作用**：不是"碰一下就行"——要至少 m 个凸位同时插入。门槛值。

### 08 · 金字塔层路由

```
┌──────────────────────────────────────┐
│ 金字塔层路由 (Pyramid + LayerPool)     │
│ 世界: 🔧 机械                        │
│ 输入: Gua (32-bit)                    │
│ 输出: 层级w (2-10), 池碰撞结果         │
│ 依赖: 榫卯(06), popcount              │
│ 状态: ⚠️ 在用但孤悬 (独立于三轴锁定)   │
│ 理论: popcount密度→分辨率层            │
│       同层碰撞 → 精度匹配             │
│ 引用: tongzi_kernel.py Pyramid.flow()│
│ 位置: 池子层                          │
│ 参数: 10层 (2-10爻), 每层2^w块砖       │
│       popcount/32*9 → 层级            │
└──────────────────────────────────────┘
```

**作用**：不让复杂字和简单字在同一层比。高密度→高层(精细判)，低密度→低层(粗判)。

**问题**：金字塔池碰撞和三轴锁定是两套独立系统。池子 `flow()` 做层内碰撞，`feed()` 做三轴匹配——互不通信。

```

│ 世界: 🌿 生物                        │
```
