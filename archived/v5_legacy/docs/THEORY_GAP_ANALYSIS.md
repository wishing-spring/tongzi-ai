# 各部件产出与输出层设计

## 一、诚实盘点：各部件能产什么

### 左脑（理性）

| 部件 | 产出 | 性质 |
|:--|:--|:--|
| 涌动池 | 碰撞激活的沉淀卦 | "这个输入和历史上哪些事相似" |
| 生态池 | 固化吸引子 | "稳定模式是什么" |
| 沉淀环 | 分桶检索命中 | "类似问题之前怎么处理的" |
| 齿轮耦合 | 跨池回路 | "不同区域间的逻辑关系" |

→ 擅长：逻辑、结构、事实关联、模式匹配

### 右脑（感性）

| 部件 | 产出 | 性质 |
|:--|:--|:--|
| 双天元 | 当前位置卦 | "系统现在心情在哪个波段" |
| 脊骨 | 轨迹序列 | "最近经历了什么波动" |
| 刚柔折叠 | past⊕future | "自我认知——我以为我在哪" |
| 引力 | 偏移方向 | "往什么感觉靠" |
| 因果贯通 | 一致性值 | "内部五流有多统一" |

→ 擅长：感觉方向、语气定位、态度偏向、自我状态

右脑不产事实、不产逻辑。纯感性引导。

## 二、输出层：两路织成文

### 左路给骨架，右路给血肉

```
左脑产出（碰撞吸引子+检索卦）→ 内容骨架: 回答什么
右脑产出（天元+引力+因果）   → 风格血肉: 怎么回答
```

具体：

```
骨架（左）: 沉淀环检索命中 → "雨天"相关卦组 → 解码 → ["天气", "阴沉", "带伞"]
血肉（右）: 天元偏暖 → 选语气模板 → "记得带伞喔" 而非 "请携带雨具"

最终: "天气阴沉，记得带伞喔"
```

骨架保证逻辑关联——回答和输入有关。血肉保证感性风格——说话像人不像机器。

## 三、解码器不是一步翻译

解码器是一个**排序+编织器**：

```
输入: 一批卦（来自不同部件）
步骤:
  1. 来自沉淀环的卦（骨架）→ 按Hamming排序 → 确定回答内容线索
  2. 来自灵犀的卦（血肉）→ 映射到语气/风格/态度
  3. 编织: 骨架做主干，血肉选措辞
  4. 连环XOR展开: 一卦一卦出字
  5. 序列检查组内XOR差 → 保证逻辑连贯
输出: 成文
```
