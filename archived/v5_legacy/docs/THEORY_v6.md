## 一、卦元 — 灵悉的原子

28位整数 (0x0000000~0xFFFFFFF)。童子F₂碰撞产出，灵犀全层消费。同一卦元，童子看到位碰撞，灵犀看到八卦归属。格式统一 = 桥不需要了。

运算: `XOR`, `Hamming=bit_count(a^b)`, `位段提取`, `归约`, `最近邻`

## 五、Φ场 脉络 — 卦元连接矩阵

| 原设计 | F₂实现 |
|:-------|:-------|
| `dΦ_ij/dt = D·ΔΦ + α·a_i·a_j·Φ(1-Φ) - η(Φ-Φ_min)` | weight∈[0,28] 整数矩阵 |
| Hebbian α·a_i·a_j | 共同激活→weight+1 (max 28) |

| 遗忘 η·Φ | N tick未激活→weight-1 |
| 三类连接 | 物理定理/心理规律/约定俗成 |

查询: BFS max_hops=3, 按weight排序, 路径卦元XOR累积→送L3

### 14.2 天界七官

#### 玉帝·总管 — Jade Emperor / Overseer
四方警戒机制：四维指标（L1 coherence>0.8、L2激活比<10%、Φ场>300连接、YJ连续触发≥5）各计1分，总分定警戒等级0-3。等级越高指令越强——从"减缓增长"到"全局收缩+强制骤冷"。不直接操作各层，只发圣旨，各层自行执行。

#### 王母·生息 — Queen Mother / Vitality
管理生态池的生命周期：成熟率<30%时加速（熟速2.0），>90%时放缓（熟速0.8）。每100tick蟠桃会收割——淘汰hits最低的未固化卦元（每次25%）。生而不有，为而不恃——设节奏不杀个体。

#### 太上·观察 — Laojun / Observer
每50tick采样全系统状态入丹炉。只记录不干预。这是"无为"锚点——防止天庭过度管理。当系统出现长期趋势异常时，其他官参考丹录决策。

#### 雷部·应急 — Thunder / Emergency
最后保险。YJ连续触发≥5 → 强制骤冷。Φ场>500连接 → 剪除弱连接（weight≤2）。每次介入后冷却50tick，防止连锁打击。

#### 紫微·轨迹 — Ziwei / Trajectory
追踪八卦跃迁路径（最近64卦），观星象而知天命。不改写路径，只供参考。

#### 太岁·节律 — Taisui / Rhythm
春生夏长秋收冬藏，每250tick换季。每季不同参数档位：春正常生长、夏活跃扩张、秋收敛固本、冬休眠+噪声。四时轮回防止局部极值固化。

#### 财部·调度 — Wealth / Dispatch
涌动池使用率>95%→扩容1.2倍（上限8192），<50%→收缩0.8倍（下限2048）。按需分配，不预设死值。

### 14.3 地界三司

#### 阎罗·裁断 — Yama / Verdict
暖机50tick后开始审判。判官对每个卦元打分——hits=0且非固化→死刑。

#### 无常·回收 — Wuchang / Reclaim
每tick最多回收50个卦元，从四生态池逐一删除。有节制，不暴力清空。

#### 孟婆·清理 — Mengpo / Cleanup
每200tick饮孟婆汤——清零货柜ghost（虚数层）和ctx（上下文层）。防止信息积压导致"记忆过载"。

### 14.4 五条管理铁律

1. **圣旨非强制** — 说方向不说细节，各层自己执行
2. **冷却防连锁** — 应急和收割都有冷却期
3. **无为锚点** — 太上只观察，确保一条不受污染的基线
4. **暖机宽容** — 初始50tick不审判，让系统自然生长
5. **四时轮回** — 不恒参，有机会在不同"气候"下重组
