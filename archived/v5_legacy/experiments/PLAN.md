# 童子 · 双组并行计划

## 一组：扩位生态组（实验场）

**负责：铜须**
**定位：实验室，不做设计，只跑实验、出数据、记成果**

| 路径 | 内容 | 状态 |
|:--|:--|:--|
| B - 32位扩展 | VEC_DIM=32，对照实验 | ⏳ 待开工 |
| A - 选择压力 | 资源竞争/稀疏奖励/闭环供能 | ⏳ 待设计 |
| 定期测试 | 80 单元测试 + 新实验验证 | 🔁 持续性 |
| 成果归档 | 实验报告 → experiments/ → README | 🔁 持续性 |

**铁律**：不动核心架构，只跑实验，记录数据。失败也记。

## 二组：架构模型改造组（主力）

**负责：胖子（军团长）牵头，携手铜须 + 外置 AI**
**定位：追求终极通用大模型，理论研究 + 架构设计**

| 阶段 | 内容 |
|:--|:--|
| 研讨 | 军团长提出方向 → 铜须 + 三 AI 交叉分析 → 形成方案 |
| 设计 | 方案上报确认 → 公式化/模型化 → 理论闭环 |
| 蓝图 | 核心蓝图定位、依赖标注、状态追踪 |
| 落地 | 理论闭环后 → 编码实现 → 测试验证 |

**铁律**：理论闭环优先于工程实现，每项必须四件套（公式 + 方法 + 途径 + 标注）。

---

## 双组协作

- 一组产出数据 → 二组参考数据做架构决策
- 二组理论突破 → 一组设计新实验验证
- 互不阻塞，平行推进
