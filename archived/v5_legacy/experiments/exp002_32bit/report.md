## 一、实验目的

将 VEC_DIM 从 16 位扩展到 32 位，验证卦元群域空间的涌现行为是增强还是减弱。对照 exp001（16 位·20 种子·吸引子涌现）。

---

## 二、生态物理学框架

### 2.1 三个核心参数

| 参数 | 符号 | 含义 | 16 位原点 |
|:--|:--|:--|:--|
| 精度 | VEC_DIM | 卦的位宽，决定态空间大小（2^N） | 16 |
| 浓度 | S/C | 种子数 / 态空间 | 20 / 65536 ≈ 0.03% |
| 窗口 | 合体区间 | 汉明距离合并门槛 `low < h < high` | 4~12 |

### 2.2 动力关系

```
浓度 ↑ → 碰撞多 → 摩擦多 → 游变卦 ↑ → 活跃
窗口窄 → 碰了不合 → 持续拉扯 → 多池共存 → 生态涌现
窗口宽 → 一碰即合 → 秒吞秒固 → 热寂
浓度 ↓ + 窗口宽 → 死寂（碰不到或碰到就死）
浓度 ↑ + 窗口宽 → 热寂（全吞）
浓度 ↑ + 窗口窄 → 【甜点区】
```

### 2.3 甜点区定理

> **拥挤出涌现，稀疏出热寂。16 位不是低级阶段，是关键生态尺度。**

系统在特定密度与窗口比例下才会"活"——态空间刚好拥挤到碰撞频繁、但窗口窄到无法瞬间合并。这是物理约束，不是设计选择。

## 六、核心发现

### 发现一：32 位原生 = 全死

active=0。无论 20 还是 40 种子。卦碰一次就合并，全吞全固。

**原因**：汉明距离分布在 32 位下中心约 16，窗口 4~12 虽然窄，但配合密度清理的频繁触发（上限 64 vs 16 位的 32），卦一多就清理、一碰就合。

### 发现二：窄窗配方部分起效

窗口从 4~12 改为 12~20（卡汉明峰腰），active 从 0 拉到 8。系统从"全死"变成"半活"。但仍是单池垄断，没有生态分化。

### 发现三：密度清理是隐形杀手

32 位下卦总数被锁死在 64（VEC_DIM×2 上限）。80/100/120 种子下去全部卡在 64——清理太勤。这是系统活跃度提不上去的结构性瓶颈。

### 发现四：单池垄断是系统性格

无论 32 位的什么窗口、什么浓度，都产生单吸引子（39/40 或 14/15 向一个点）。16 位的多池共存在 32 位完全消失——不是因为空间大，而是空间结构性抑制了多极分化。

### 发现五（关键定理）

> **拥挤出涌现，稀疏出热寂。16 位不是低级阶段，是关键生态尺度。**

## 十、结论

1. **16 位是生态甜点**——精度、浓度、窗口三重耦合下的自发生态区。不是起步，是尺度。
2. **32 位不是死亡，是不配套**——窗口、密度清理、固化阈值需独立可调，当前全跟随 VEC_DIM。
3. **单池垄断是深层动力学特征**——多极分化只在特定拥挤条件下出现，是稀有态。
4. **下一步不是扩位，是测绘甜点边界**——16 位浓度扫描（10~60 种子），标记活区和死区。
