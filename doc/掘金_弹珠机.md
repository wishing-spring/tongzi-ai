# 弹珠机编码：16根XOR柱子，零训练实现汉字语义聚类

## TL;DR

用三台"弹珠机"（bit-gated XOR 累加器）替代词嵌入，在纯 F₂ 空间实现汉字语义保距编码。31字测试准确率 83%，随机基线 25%。全过程零浮点、零训练、纯位运算。

---

## 1. 动机

词嵌入需要训练、浮点运算、不可审计。我们想看看：**能不能用纯位运算做到同样的事？**

约束：GF(2) 原生、零外部依赖、可追溯因果链。

## 2. 弹珠机编码

### 2.1 核心操作

```python
def pinball(cp, pins):
    v = cp
    for i in range(16):
        if cp & (1 << i):
            v ^= pins[i]     # 撞柱
    return (v | (v << 16)) & 0xFFFFFFFF
```

16 根柱子，每根是一个 32 位常量。码点的每个 1-bit 决定是否 XOR 对应的柱子。

### 2.2 为什么保距

弹珠机**不是线性映射**。`h(a⊕b) ≠ h(a)⊕h(b)`。

码点差几个 bit，就走岔几条柱子路径。差 2 bit → 14 根同路 → 结果相近。差 4 bit → 12 根同路 → 结果偏移更大。

### 2.3 三轴独立证人

```python
A(ch) = pinball(cp, P_A)  # 质数柱
B(ch) = pinball(cp, P_B)  # 平方柱
C(ch) = pinball(cp, P_C)  # 位移柱
```

三组不同柱阵 → 三个独立判断 → 三轴同时 ≥ 阈值才算锁定。

## 3. 内生因果机

弹珠机编码嵌入因果时序：

```
编码 → 金字塔池（按 popcount 分层）
     → 活性窗口 τ=5
     → 三轴锁定 m=9
     → 归约 / 剪枝
```

- **锚定态**：已固化历史卦元
- **活性态**：悬挂窗口内，有 5 次匹配机会
- **结果态**：锁定成功 → 归约；超时 → 剪枝
