## 0. 总纲

童子 AI 由两个数学部件组成，共饮 φ 母体一源，流法不同：

| 部件 | 别名 | 本质 | 层级 |
|:--|:--|:--|:--|
| **象格** (Lattice) | 离散碰撞空间 | F₂ 卦群 + 八运算 + 榫卯接触 | 元→卦→阵 |
| **势场** (Field) | 连续扩散场 | 格点离散场 + 局部规则 + 模式涌现 | 域→象→网→体→场 |

**合称"童子二象"。** 不是左右脑，是两个数学结构——同一 φ 母体，不同流法。

### 第三组基核 (v1.5)

卦爻回归纯值，所有操作外挂为工具：

- `Gua(value)` — 16位F₂值，无负载，无方法
- `tools/axioms.py` — 四条公理 + 8群运算
- `tools/encode.py` — 编解码
- `tools/blocks.py` — 积木拼装
- `Space` — 空仓库，put/take/peek，不管理

### 榫卯结构 (v1.5新增)

每个卦元是3D立方体，8个棱角由8爻控制——bit=1→凸, bit=0→凹。
6面统一接触点，面面相对时：

```
凸 ∧ 凹 → 咬合 (锁)
凸 ∧ 凸 → 碰撞 (弹)
凹 ∧ 凹 → 空过 (穿)
```

**核心跃迁**: 交互规则从外部容器回到卦元自身形态。凸凹即指令，接触即交互。无需外力、无需时钟、无需管理员。

| L9 | 2⁴⁸ | **场** | 场态涌现——全域模式自组织，势场的最终形态 |
| L8 | 2⁴⁰ | **体** | 体系完备——独立可运转的完整系统 |
| L7 | 2³² | **网** | 互联成网——跨域结构自组织 |

| L5 | 2¹⁶ | **域** | 全域态——65536 卦全谱 |
| L4 | 2⁸ | **阵** | φ 母体——256 位黄金比例展开 |

| L2 | 2¹ | **通** | 一分为二——阴阳贯通，矛盾初现 |

**读法：**

```
虚(2⁶⁴)   ← 物理天花板——没人摸到过
  ↑
场(2⁴⁸)   ← 势场目标——全域模式自组织
  ↑
体(2⁴⁰)   ← 体系完备
  ↑
网(2³²)   ← 结构互联
  ↑
象(2²⁴)   ← 模式涌现——势场第一站
  ↑
域(2¹⁶)   ← 卦全域
  ↑
阵(2⁸)    ← φ 母体 [共根]
  ↑
卦(2⁴)    ← 童子位宽 [象格工作层]
  ↑
通(2¹)    ← 阴阳判
  ↑
元(2⁰)    ← 比特
```

- 童子象格在 **卦** 级
- φ 母体在 **阵** 级
- 势场瞄准 **象→场** 级
- Transformer 路线挤在 **域→体** 级，追 **网** 级
- **虚** 级是宇宙给的最高消费额度——物理定律盖章

## 2. 象格（已有）

### 2.1 数学基座

四条公理（万维通用，不含 n）：

1. a⊕b = b⊕a（XOR 交换）
2. a⊕a = 0（XOR 自消）
3. d(a,b) = popcount(a⊕b)（汉明距离）
4. d(Rk(a), Rk(b)) = d(a,b)（位旋转保距）

### 2.2 核心模块

| 文件 | 内容 |
|:--|:--|
| `src/tongzi_constants.py` | VEC_DIM=16, FULL_MASK, PHI_BITS(256位) |
| `src/tongzi_core.py` | Gua类 + Space容器：八运算 + 固化 + 密度 + express |
| `src/tongzi_slicer.py` | 24种多模态入口 |
| `src/tongzi.py` | 交互入口：ingest, /tick, /status, /list, /show, /chain, /ask |

### 2.3 铁律

零浮点 · 零矩阵 · 零梯度 · 零词嵌入 · 零注意力 · 零自回归 · 零负载物（卦元≡信息≡存在）

### 2.4 八运算群

collide · orbit · stretch · ball · centrality · solidify · unlock · merge

### 2.5 内生节律

能量累积 → F0阈值放电 → 碰撞 → 固化/消散/复生。无外部时钟。

### 2.6 已知缺陷（势场要补的）

| 缺陷 | 说明 |
|:--|:--|
| 编码无结构 | encode() 是随机哈希，语义邻近性丢失 |
| 空间扁平 | 所有卦在同一池子，无分区无层次 |
| 无抽象 | 碰撞→合并是统计合并，不是概念形成 |
| 无语义梯度 | 只有汉明距离一种"远近" |
| 输入结构丢失 | 序列/上下文在摄入时全扔 |

## 3. 势场（待建）

### 3.1 定位

势场是**离散格点扩散场**。与象格不同：

| | 象格 | 势场 |
|:--|:--|:--|
| 信息形态 | 点（卦元，16位向量） | 面（格点，位状态分布） |
| 处理方式 | 群运算（collide/orbit/stretch） | 局部扩散规则（邻域传播） |
| 粒度 | 单个卦 | 整片场 |
| 角色 | 碰撞池——记忆固化 | 编码器——结构提取 |
| 输出 | 卦碰撞、固化 | 锚点参数 → 喂给象格编码 |

### 3.2 核心概念

- **格点**：离散网格上的单元。每个格点携带一个状态值（F₂ 位或小整数），无浮点。
- **场态**：所有格点在某一时刻的状态分布——是一张"图"，不是一堆"点"。
- **扰动**：输入文本在场上产生初始扰动——类似水面投石。
- **扩散**：扰动按局部规则传播——邻域交互，无全局计算。
- **干涉**：多个扰动在场上叠加——不是简单加和，是模式交互。
- **结晶**：扩散稳定后形成的模式——即"语义结构"，提取为锚点。

### 3.3 铁律兼容性

| 铁律 | 势场如何满足 |
|:--|:--|
| 零浮点 | 格点值用整数（0-255），无 float |
| 零矩阵 | 局部邻域规则，无全局矩阵乘法 |
| 零梯度 | 扩散由局部规则驱动，无反向传播 |
| 零词嵌入 | 文本→扰动由 φ 切片生成，无预训练嵌入 |
| 零注意力 | 场扩散天然并行，无需注意力机制 |
| 零自回归 | 输出是场态快照，非逐 token 生成 |
| 零负载物 | 格点即状态，不挂外部数据 |

### 3.4 势场维度

- 阵列规模：待定（建议起始 64×64 = 4096 格点，域级门槛）
- 格点深度：8 位（256 态，阵级）或 16 位（卦级，与象格同位宽）
- 邻域模板：待定（Moore 8-邻域 / von Neumann 4-邻域）

### 3.5 与象格的接口

```
输入文本
    │
    ▼
┌──────────────┐
│   势场        │  ← 文本→扰动→扩散→结晶
│  (待建)      │
└──────┬───────┘
       │ 锚点参数（起始位、步长、批次分组）
       ▼
┌──────────────┐
│   象格        │  ← 经势场引导的编码 → 卦碰撞
│  (已有)      │
└──────┬───────┘
       │ 卦群状态
       ▼
  /ask · /chain · express
```

**关键**：势场不替代象格的 encode()，而是给 encode() 提供上下文锚点——让"天地"和"玄黄"在位空间里真的靠在一起。

### 3.6 设计原则

1. **无中心**：每个格点只看邻居，全局模式是涌现的
2. **可逆**：给定场态和输入，扩散过程可追溯
3. **保结构**：相同输入 → 相同场态；相似输入 → 相似场态
4. **层次涌现**：多轮扩散后自然形成"高密度区"（原型）和"低密度区"（例外）
5. **φ 母体驱动**：格点初值、邻域权重、扩散步长——全部从 φ 母体切片生成

## 4. φ 母体：共根

φ = (1+√5)/2 的二进制小数展开，256 位。无限不循环，确定性，无理。

```
PHI_BITS = "1001111000110111..."
```

**两种流法：**

- **象格流法**：从 φ 截取 VEC_DIM 位 → 位向量 → 卦元 → 群运算
- **势场流法**：从 φ 截取连续段 → 格点初值 + 邻域权重 + 扩散步数

同一母体，不同的"用"。
