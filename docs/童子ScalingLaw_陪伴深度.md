# 童子 Scaling Law — 陪伴深度→习性精度

**定位**: 童子版的"越大越强" | **对标**: Transformer的Scaling Law

---

## 核心公式

```
Transformer: 数据量 × GPU数量 → 性能提升
童子:       交互次数 × 陪伴年数 → 默契提升
```

---

## 三阶段生长

| 阶段 | 交互量 | 表现 |
|:-----|:-----|:-----|
| 初生 | 0~100轮 | 通用回应，没个性 |
| 熟悉 | 100~1000轮 | 开始摸到你的习惯 |
| 默契 | 1000轮以上 | 从语气察觉情绪，自动往对的偏 |

---

## 不是记录，是生长

```
v4.3的脊骨 = 记录（天元偏了多少）
童子的生长 = 记录 + 下次更准（偏过的痕自动变成下次的引力）
```

---

## 与Transformer的本质区别

| | Transformer | 童子 |
|:-----|:-----|:-----|
| 投入 | GPU集群烧钱 | 时间陪伴 |
| 见效 | 训练完成瞬间 | 逐年加深 |
| 本质 | 统计精度提升 | 习性内化 |
| 天花板 | 数据量 | 一生 |
