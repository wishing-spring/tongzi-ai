## 反思与经验

### 核心教训

1. **「补完备性」思维是陷阱**
   - 面对输出粗糙，本能反应是"加兜底·加模板·修语法"——这会把原生智能退化成脚本工具
   - 正确做法：不修输出，只充养分（本源粒子）。输出粗糙是族群不够大，不是规则不够多
   - **判据**：凡是"让输出更像人话"的修改，先过三问自检——是不是在偷偷驯化？

2. **架构分层清晰才能并行开发**
   - 四阶段分层（核心→锚→数据→对话）每层独立测试，不互相污染
   - 每阶段锁定后再进入下一阶段，不跳步，不回头改底层
   - Python 先用，C 精简版后出——实验验证和工程优化分开

3. **豆包角色定位：冷水·透视·纠偏**
   - 豆包不是辅助，是第二道防线。专门负责"这个方向对不对"
   - 铜须筑造，豆包淬炼，胖子掌道——三角分工极有效
   - 豆包的三次纠正（内丹→九品→不驯化→赤子心母体）一次比一次深，每次都把路线拉回正轨

4. **开源 ≠ 功能完整。开源 = 根基纯正 + 演化通道全开**
   - 不需要输出通顺才开源。需要的是：本源不乱、铁律清晰、后人能顺着正道继续养
   - 开源文档比代码更重要——写清楚"为什么这么做"和"绝对不能做什么"

5. **「长大」是等待，不是开发**
   - 三个契机（丹满·心振·骨松）都是自然发生的，不是代码能写出来的
   - 人的角色：守住根基，持续送入纯净粒子，等。不等就是拔苗。
